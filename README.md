# Информационная система электронной коммерции для малого бизнеса

Сервис для автоматизации документооборота малого бизнеса. Позволяет вести каталог товаров, создавать заказы и автоматически генерировать 
документы по сделке. Документы формируются в PDF/DOCX из шаблонов. Есть REST API для интеграции с внешними системами.
Основная цель — избавить предпринимателя от ручного заполнения шаблонов в Word и Excel, сократить время на подготовку документов.

| Слой | Технология |
|------|------------|
| **Бэкенд** | Python, Django, Django REST Framework |
| **Аутентификация** | Simple JWT |
| **База данных** | PostgreSQL |
| **Генерация документов** | python-docx (docx), WeasyPrint (pdf) |
| **Экспорт** | openpyxl (Excel) |
| **Документирование API** | drf-spectacular (Swagger) |
| **Фронтенд** | React (Vite), React Router, Axios |
| **Тестирование** | pytest |
| **Инфраструктура** | Docker, docker-compose, Git |

## Похожий опыт (что уже делал)

- **Фронтенд маркетплейса на React** — полноценный проект, похожий на Авито: карточки товаров, фильтры, личный кабинет, корзина, отчеты о заказах.
- **Модуль электронного документооборота на Python** — была попытка, но пошло тяжело. Код получался запутанным, логика давалась сложно. В итоге не доделал.

## Мои риски

### Риск №1: Бэкенд в целом (Python, Django)
**Проблема** Давно не писал на Python, бэкенд даётся тяжело. Не люблю и плохо понимаю, как правильно выстраивать логику на серверной стороне.

**Предположительное решение**
- Иду строго по плану ТЗ, не выдумываю лишнего функционала

### Риск №2: Docker с нуля
**Проблема** Никогда не работал с Docker. Не понимаю, как собирать образы, настраивать docker-compose.

**Предположительное решение**
- На 1–2 неделе Docker не трогаю вообще — разрабатываю локально
- На 3-4 неделе пытаюсь разобратся через гайды и практику

### Риск №3: Генерация документов (python-docx, WeasyPrint)
**Проблема** Полный ноль. Не представляю, как заполнять шаблоны.

**Предположительное решение**
- На 1-й неделе провожу технический спайк: пробую создать простой docx-шаблон, заполнить его данными из кода, сгенерировать PDF
- Если WeasyPrint не пойдёт с кириллицей — перехожу на ReportLab
- Делаю минимально жизнеспособный вариант: один шаблон, одна кнопка «Скачать PDF»
